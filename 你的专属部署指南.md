## 📝 部署步骤（只需 5 分钟）

### 第 1 步：创建 Pull Request（合并代码）

1. **打开浏览器，访问你的仓库：**
   ```
   https://github.com/Lxc-0813/cabin_game
   ```

2. **你会看到一个黄色提示框**，类似这样：
   ```
   claude/add-local-multiplayer-VUX2T had recent pushes
   [Compare & pull request] 按钮
   ```

3. **点击绿色的 "Compare & pull request" 按钮**

   如果没看到这个提示框，手动操作：
   - 点击仓库页面上方的 "Pull requests" 标签
   - 点击绿色的 "New pull request" 按钮
   - Base 选择：`main`
   - Compare 选择：`claude/add-local-multiplayer-VUX2T`
   - 点击 "Create pull request"

4. **填写 Pull Request 信息**
   - Title（标题）：`添加局域网联机功能`
   - Description（描述）：可以留空或写点说明
   - 点击绿色的 "Create pull request" 按钮

5. **合并 Pull Request**
   - 点击绿色的 "Merge pull request" 按钮
   - 点击 "Confirm merge" 确认
   - ✅ 合并完成！

### 第 2 步：启用 GitHub Pages

1. **点击仓库顶部的 "Settings"（设置）标签**
   - 在仓库页面的右上方，工具栏最右边

2. **在左侧菜单中找到 "Pages"**
   - 往下滚动左侧菜单
   - 点击 "Pages"

3. **配置 Pages**
   - **Build and deployment** 部分
   - **Source** 下拉菜单选择：**GitHub Actions**
   - 不要选 "Deploy from a branch"！一定要选 "GitHub Actions"

4. **保存**
   - 选择后会自动保存
   - 不需要点击其他按钮

### 第 3 步：等待自动部署

1. **回到仓库首页**
   - 点击仓库名 "cabin_game" 回到首页

2. **查看部署状态**
   - 点击顶部的 "Actions" 标签
   - 你会看到一个工作流正在运行（黄色圆圈⚠️）
   - 等待变成绿色的勾 ✅（大约 2-3 分钟）

3. **查看部署详情（可选）**
   - 点击正在运行的工作流
   - 可以看到实时日志

### 第 4 步：访问你的游戏！

部署成功后，打开浏览器访问：

## ⚠️ 重要提示

### 网页版的限制

**只能单机玩！** 网页版不支持局域网联机，因为：
- GitHub Pages 只能托管静态网页
- 不能运行 WebSocket 服务器
- 联机功能只能在本地使用

### 如果部署失败

1. **检查 Actions 标签页**
   - 看看有没有红色的 ❌
   - 点击查看错误日志

2. **常见问题：**
   - 构建失败 → 检查代码是否有错误
   - 404 Not Found → 等待 5-10 分钟
   - 空白页面 → 检查 vite.config.ts 的 base 路径

3. **重新部署**
   - 在 Actions 标签页
   - 点击失败的工作流
   - 点击右上角的 "Re-run jobs"
